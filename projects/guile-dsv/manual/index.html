1 Introduction



Guile-DSV is a module that provides procedures for working with
delimiter-separated values (DSV) format for programs written in
GNU Guile.



Guile-DSV supports both
the Unix-style DSV format and RFC 4180.

Guile-DSV sources are available from GitHub at
https://github.com/artyom-poptsov/guile-dsv/.  This section
describes requirements of Guile-DSV and installation process.



Guile-DSV depends on the following packages:



    

  •  GNU Guile, 2.2+

  •  Guile Library, 0.1.9+

  •  Guile-SMC, 0.6.2+

2.1 GNU Guix



On GNU Guix the package can be installed as
follows:





$ guix install guile-dsv




If you want to install the most recent version of Guile-DSV right from the
“master” branch (on your own risk), you can use the following set of
commands:





$ git clone git@github.com:artyom-poptsov/guile-dsv.git
$ cd guile-dsv
$ guix package -f guix.scm

2.2 Manual



On Ubuntu GNU/Linux, you will need the following packages:



$ sudo apt install \
    make \
    automake \
    autoconf \
    guile-3.0 \
    guile-3.0-dev \
    guile-library \
    texinfo \
    help2man




Get the sources of Guile-DSV from GitHub using Git (a good introduction to Git
is Pro Git book, which is available
online):





$ git clone git@github.com:artyom-poptsov/guile-dsv.git




Configure the sources:





$ cd guile-dsv/
$ autoreconf -vif
$ ./configure --prefix=/usr




Build and install the library:





$ make -j$(nproc)
$ make install




For a basic explanation of the installation of the package, see the
‘INSTALL’ file.



Please note that you will need
Automake version 1.12 or
later to run self-tests with make check (but the library
itself can be built with older Automake version such as 1.11).



important You probably want to call configure with
the ‘--with-guilesitedir’ option so that this package is
installed in Guile’s default path.  But, if you don’t know where your
Guile site directory is, run configure without the option,
and it will give you a suggestion.

3 API Reference



These procedures are provided by (dsv) module.



All the procedures accept an optional format argument that specifies the
format of DSV data.  Allowed formats are the following symbols:





‘unix’


the Unix-style DSV format



‘rfc4180’


RFC 4180






By default a format is set to 'unix.





Scheme Procedure: dsv->scm [port [delimiter]] [#:format='unix] [#:comment-prefix='default] [#:debug-mode?=#f] [#:log-driver=#f] [#:log-opt='()]


Scheme Procedure: dsv-string->scm string [delimiter] [#:format='unix] [#:comment-prefix='default] [#:debug-mode?=#f] [#:log-driver=#f] [#:log-opt='()]


Read DSV data in a specified format from a port.  If the
port is not set, read from the default input port.  If a delimiter
is not set, use the default delimiter for the format.



When a format is set to 'unix, lines commented with a
comment-prefix will be skipped.  comment-prefix is expected to be
either a string or 'default symbol (which means to use the default
prefix for the format), or 'none symbol (that tells the procedure not
to skip anything); default value of comment-prefix for Unix format is
“#”.



Empty lines in DSV data will be converted to lists that contain one element
(an empty string.)



When format is set to 'unix, then following nonprintable characters
will be escaped with backslashes: #\page, #\newline,
#\return, #\tab, #\vtab.  E.g. #\page will become
“\f”.



The procedure returns a list of lists that represents a DSV table, or throws
an error.



log-driver allows to set the logging driver. Possible values are:
“syslog” (used by default when log-driver is set to #f),
“file”, “null”.  Using log-opt parameter one can pass additional
options for a logging driver.



The dsv-string->scm procedure works the same way as dsv->scm,
except it takes a DSV string instead of a port.



Examples:





(dsv->scm (open-input-file "/etc/passwd"))
⇒
(...
 ("news" "x" "9" "13" "news" "/usr/lib/news" "/bin/false")
 ("root" "x" "0" "0" "root" "/root" "/bin/zsh"))

(dsv-string->scm "a:b:c")
⇒ '(("a" "b" "c"))

(dsv-string->scm "a;b;c" #\;)
⇒ '(("a" "b" "c"))

(dsv-string->scm "a,b\\,c" #\,)
⇒ '(("a" "b,c"))

;; Handle backslash escapes:
(dsv-string->scm "a\\nb\\\\c\n")
⇒ '(("a\nb\\c"))

;; Handle record continuation by ignoring backslash-escaped newlines:
(dsv-string->scm "a \\\nb:c")
⇒ '(("a b" "c"))










Scheme Procedure: scm->dsv list [port [delimiter]] [#:format='unix]


Scheme Procedure: scm->dsv-string list [delimiter] [#:format='unix]


Write a list of values list as a sequence of DSV strings in a specified
format to a port.  If the port is not set, write to the
default output port.  If a delimiter is not set, use the default
delimiter for a format.



The procedure accepts either a DSV table (as a two-dimensional list), or
a single table row (as a one-dimesional list of strings).



When format is set to 'unix, then the following escaped nonprintable
characters will be accepted: #\page, #\newline, #\return,
#\tab, #\vtab.  E.g. “\f” will be read as #\page.



Return value is undefined.  The procedure throws a dsv-parser-error on
an error.



The scm->dsv-string procedure works the same way as scm->dsv,
except it returns DSV data as a string instead of printing it to a port.



Examples:





(scm->dsv '("news" "x" "9" "13" "news" "/usr/lib/news" "/bin/false"))
-| news:x:9:13:news:/usr/lib/news:/bin/false

(scm->dsv '(("news" "x" "9" "13" "news" "/usr/lib/news" "/bin/false")
            ("root" "x" "0" "0" "root" "/root" "/bin/zsh")))
-|
news:x:9:13:news:/usr/lib/news:/bin/false
root:x:0:0:root:/root:/bin/zsh

(scm->dsv-string '("a" "b" "c"))
⇒ "a:b:c\n"

;; If a delimiter occures inside a field, it will be escaped with a backslash:
(scm->dsv-string '("a" "b:c"))
⇒ "a:b\\:c\n"

;; Use a custom field delimiter:
(scm->dsv-string '("a" "b" "c") #\,)
⇒ "a,b,c\n"

;; Make use of RFC 4180:
(scm->dsv-string '("a" "b" "c,d") #:format 'rfc4180)
⇒ "a,b,\"c,d\"\r\n"

;; Escape non-printable characters and backslashes:
(scm->dsv-string '("a\nb" "c\\d"))
⇒ "a\\nb:c\\\\d\n"










Scheme Procedure: guess-delimiter string [known-delimiters] [#:format='unix]


Guess a DSV string delimiter.  Optionally accept a list of
known-delimiters as an argument.  Default known delimiters are:
#\,, #\:, #\;, #\|, #\tab, #\space.



The procedure returns #f if it cannot determine a delimiter based on
the given arguments.  If an error occured then it throws
dsv-parser-error.



Note that when known-delimiters list contains less than two elements,
the procedure returns #f.



Examples:





(guess-delimiter "a:b,c,d")
⇒ #\,

(guess-delimiter "a,b:c")
⇒ #f

(guess-delimiter "a,b,c" #:format 'rfc4180)
⇒ #\,

;; Using custom known delimiters list:
(guess-delimiter "a-b-c" '(#\- #\,))
⇒ #\-

3.1 Tables



Module (dsv table) provides procedures for working with parsed DSV
tables.





Scheme Procedure: string* str k


Multiply a string str by k in a Python way to construct a new
string from Kth instances of str.  Return the new string.






Scheme Procedure: stylize str style


Use a style terminal escape codes to change the style of a string
str.  Return the styled string with escape codes.






Scheme Procedure: table-print-element element port


Print an element of a table to a port.  If an element is
#f, then print a single space.






Scheme Procedure: print-table-parameters port


Print all known table parameters to a port.






Scheme Procedure: shorthand->table-parameter sh


Convert a shorthand sh to a table parameter.






Scheme Procedure: get-width table


Get maximum field width for each row of table.






Scheme Procedure: format-table table borders [#:with-header?=#f] [#:port=(current-output-port)]




Print a formatted table to a port, using borders
specification.






Scheme Procedure: table-map proc table


Apply a procedure PROC to each cell of a table.  Return a new
table.






Scheme Procedure: table-filter-row proc table


Remove all the rows from a table for which procedure proc returns
#f.






Scheme Procedure: table-filter-column proc table


Remove all the columns from a table for which procedure proc
returns #f.






Scheme Procedure: table-wrap table current-column-widths [#:width=80] [#:padding=0]




Wrap a table in such way that the table will fit into width
columns.



current-column-widths should be the value returned by get-width
procedure.








Scheme Procedure: table-wrap-row row widths


Wrap a table row to fit each cell it into the specified widths.
Return a list where each row is represented as a sub-list of strings.






Scheme Procedure: string-slice s width


Slice a string s into parts of width length.  Return the list of
strings.

3.2 Table Presets



Module (dsv table-preset) provides procedures for working with table
presets.  A table preset is a pre-defined table style that specifies table
outer borders, row separators etc.





Scheme Procedure: print-table-presets port


Print all known table presets to a port.






Scheme Procedure: load-table-preset name


Read table preset name from a file.  Return the preset as a list.






Scheme Procedure: table-preset-override preset borders


Override a table preset with a borders specification.  Return the
updated table preset.






Scheme Procedure: table-preset-name? string


Check if a string is a table preset name.

3.3 Version Information



Module (dsv version) contains procedures that allow to acquire
information about the Guile-DSV version in the Semantic Versioning format.



Guile-DSV version consists of three parts:



   MAJOR.MINOR.PATCH




The procedures below allow to get any of the version part or the version as a
whole.





Scheme Procedure: dsv-version/string


Get the raw Guile-DSV version as a string.






Scheme Procedure: dsv-version


Return the Guile-DSV version as a list of the following form:
'(MAJOR MINOR PATCH)






Scheme Procedure: dsv-version/major


Scheme Procedure: dsv-version/minor


Scheme Procedure: dsv-version/patch


Get the MAJOR, MINOR or PATCH parts of the Guile-DSV version.
